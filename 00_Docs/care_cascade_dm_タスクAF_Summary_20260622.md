## この研究は何のためにやったのか

日本では「NDB（ナショナルデータベース）」というほぼ全国民の医療情報を集めたデータベースが公開されている。糖尿病に関連する指標として、特定健診での血糖スクリーニング値・HbA1c検査の算定回数・合併症管理プログラムの患者数・抗糖尿病薬の処方量など、複数の種類のデータが利用できる。

**問い：これらの指標は「同じ糖尿病の地域負担」を測っているのか？**

研究者や政策立案者が「A県は糖尿病が多い」と判断するとき、どの指標を使うかによって全く異なる答えが出る可能性がある。本研究はこの問いを、日本全国約335か所の「二次医療圏」単位で体系的に検証する。

今回の作業（タスクA〜F）は、**論文草稿の完成まで**を一気通貫で実施したものである。

| タスク | 内容 |
|--------|------|
| **A** | NDBデータから5種類の指標を抽出・整理する |
| **B** | 地域ごとの年齢の偏りを補正する（年齢標準化） |
| **C** | 指標どうしの関係を統計・グラフで可視化する |
| **D** | 結果が偶然や特定の地域に依存しないか確認する（頑健性解析） |
| **E** | 英語論文の草稿を作成する |
| **F** | 査読者からの想定質問への回答メモを準備する |

## タスクA：データ抽出（NDB Open Data No.11 から指標を取り出す）

### やったこと

厚生労働省が公開しているNDB Open Data No.11（FY2024レセプト・FY2023特定健診）の Excel ファイルから、糖尿病に関連する5種類の指標を、全国約335の二次医療圏ごとに抽出した。

**主解析で使う5指標**

| 指標 | データ種別 | 内容 |
|------|----------|------|
| HbA1c高値率 | 特定健診（FY2023） | HbA1c 6.5%以上の割合 |
| FPG異常率 | 特定健診（FY2023） | 空腹時血糖 110mg/dL以上の割合 |
| HbA1c検査算定回数 | レセプト（FY2024） | 医療機関でのHbA1c検査件数 |
| B001-20 患者数 | レセプト（FY2024） | 糖尿病合併症管理料の算定患者数 |
| B001-27 患者数 | レセプト（FY2024） | 糖尿病透析予防指導管理料の算定患者数 |

**なぜ処方薬データを主解析から外したか（重要）**

NDB No.11 の処方薬データ（糖尿病剤：薬効コード396）は**都道府県（47か所）単位のみ**で公開されており、二次医療圏（335か所）単位のデータは存在しない。このため、処方薬指標は「頑健性の確認」（タスクD）でのみ使用し、主解析（335圏）からは除外した。この理由は報告書・論文の補足資料に明記している。

### 結果

| 出力データ | 圏数 | 欠損（秘匿）率 |
|-----------|------|--------------|
| HbA1c高値率 | 335 | 0% |
| FPG異常率 | 335 | 0% |
| HbA1c検査算定回数 | 335 | 0% |
| B001-20 患者数 | **290** | **13.4%** |
| B001-27 患者数 | **291** | **13.1%** |

B001-20とB001-27は農村部を中心に「10件未満のため非公開（秘匿）」となっている圏が約13%あった。これらは欠損値として処理し、補完は行わなかった（研究倫理上の規定）。

## タスクB：年齢標準化（地域の年齢構成の偏りを補正する）

### やったこと

都道府県によって「高齢者が多い地域」「若い世代が多い地域」という差があるため、単純に指標を比較すると年齢構成の違いが結果に影響してしまう。この影響を取り除くため「年齢標準化」を行った。

- **健診指標**（HbA1c高値率・FPG異常率）：2020年国勢調査の40〜74歳人口（7つの5歳階級）を基準として、**直接法**で標準化した。
- **臨床指標**（HbA1c検査・B001-20・B001-27）：NDB公開データには医療圏別の年齢内訳がないため直接法が使えず、健診受診者数を「代理の分母」として1,000人あたりの件数に換算した。

### 結果

| 指標 | 補正前（平均） | 補正後（平均） | 変化の方向 |
|------|-------------|-------------|----------|
| HbA1c高値率 | 7.66% | 7.55% | わずかに低下 |
| FPG異常率 | 15.53% | 16.28% | わずかに上昇 |

全国335圏のデータセットが完成した。

## タスクC：相関行列・PCA・図表作成（指標どうしの関係を「見える化」する）

### やったこと

5指標の間で「どの指標とどの指標が似た動きをするか」をSpearman相関係数で調べた。また、5指標をまとめてどう整理できるかを**主成分分析（PCA）**で解析した。

### 結果：相関行列

5指標を全ペア（10通り）で比べた相関係数の一覧。

| | HbA1c高値率 | FPG異常率 | HbA1c検査 | B001-20 | B001-27 |
|--|:---------:|:--------:|:--------:|:-------:|:-------:|
| **HbA1c高値率** | — | 0.32*** | 0.04 | **−0.07** | 0.15** |
| **FPG異常率** | | — | 0.43*** | −0.23*** | −0.04 |
| **HbA1c検査** | | | — | 0.00 | −0.01 |
| **B001-20** | | | | — | 0.43*** |
| **B001-27** | | | | | — |

*\*\*\*p<0.001, \*\*p<0.01, \*p<0.05*

**最も重要な発見**：HbA1c高値率（健診スクリーニング）と B001-20（合併症管理）の相関係数は **ρ = −0.07（ほぼゼロ）**。「HbA1cの高い患者が多い地域で合併症管理プログラムが活発に使われているわけではない」ことを示している。

### 結果：PCA（主成分分析）

5指標を整理すると、**2つの大きな軸**に分かれることがわかった。

| 主成分 | 分散説明率 | 主にどの指標が高い？ | 解釈 |
|--------|---------|------------------|------|
| **PC1** | **31.2%** | FPG異常率・HbA1c検査・HbA1c高値率 | **「診断・検査強度」** の軸 |
| **PC2** | **24.0%** | B001-20・B001-27 | **「合併症管理強度」** の軸 |
| PC1+PC2 合計 | **55.2%** | | 5指標は「1つの糖尿病負担」に収束しない |

外部検証として「高齢化率」との相関を確認したところ、PC1は高齢化率が高い地域ほど高く（スクリーニングが活発）、PC2は高齢化率が低い（都市部）ほど高い（管理体制が整備されている）という結果が得られた。

## タスクD：頑健性解析（結果が偶然ではないことを確認する）

### やったこと

主要な発見（特に「HbA1c高値率 × B001-20 の無相関」）が、解析の条件を変えても変わらないかを4つの方法で確認した。

### 結果

**D1：解析単位を「二次医療圏（335か所）」から「都道府県（47か所）」に変えたらどうなるか**

10ペア中**8ペア**で、両スケールでの相関係数の差が0.15未満 → 結果は概ね安定していた。

**D2：外れ値になりやすい地域（東京・大阪・沖縄）を除いたらどうなるか**

5つのシナリオで除外を変えて確認したところ：
- HbA1c高値率 × FPG異常率 など3ペアは全5シナリオで有意 → 頑健
- **HbA1c高値率 × B001-20 は全5シナリオで非有意** → Null（無相関）の結果が頑健

**D3：処方薬データを加えたらどうなるか（都道府県レベルのみ）**

| ペア | 相関係数 | 有意水準 |
|------|---------|---------|
| 処方薬 × HbA1c検査 | **0.87** | *** |
| 処方薬 × FPG異常率 | 0.53 | *** |
| 処方薬 × B001-20 | −0.20 | n.s. |

処方薬はスクリーニング系指標と強く相関し、管理系（B001-20）とは無関連 → 主解析の構造と一致。

**D4：解析単位の大きさによる歪み（MAUP）の評価**

Low risk（歪み小）: 6/10 ペア、Moderate: 2/10、High: 2/10

## 発生した問題と解決策

| # | 問題 | 原因 | 解決策 |
|---|------|------|--------|
| 1 | マージ後のデータ行数が5,532行になった（正しくは335行） | 「西部」「東部」など複数の都道府県に同名の地区があり、地区名でマージすると重複が爆発した | マージキーを**地区コード（4桁数字）**に変更 |
| 2 | Excelから読み込んだ地区コードが「101.0」（小数）になる | ExcelがコードをFloat型で保存していた | `str(int(code)).zfill(4)` で文字列4桁に統一 |
| 3 | 出力データが335行でなく350行になった | 沖縄県のコード4705（八重山）に「二次医療圏判別不可」行が混入し、マージ時に4行に増殖した | 「判別不可」行を除去してから重複を解消する処理を追加 |
| 4 | 図の日本語が文字化けする可能性（警告） | matplotlibのデフォルトフォントが日本語未対応 | 全図のラベルを**英語表記**に統一（都道府県名もローマ字に変換） |
| 5 | QMDレンダリング時にエラー（`reference-doc: null`） | QuartoのYAMLスキーマは文字列を要求するためnullが不正 | `reference-doc: null` 行を削除 |
| 6 | QMDレンダリング時に`vancouver.csl`が見つからない | CSLファイルが`04_Manuscripts/`に存在しなかった | 他プロジェクトからコピーして配置 |
