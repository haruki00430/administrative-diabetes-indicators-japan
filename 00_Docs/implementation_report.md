## 3. 処方薬データを二次医療圏解析から除外した理由（必須記載事項）

### 除外理由

NDB オープンデータ No.11 において、抗糖尿病薬（薬効分類コード 396、糖尿病剤）の処方数量データは**都道府県別集計のみ**が公開されており、**二次医療圏別の集計ファイルは存在しない**。

具体的には、NDB No.11 の「05_処方薬」フォルダ内の公開ファイルは以下の構成となっている:
- `【内服】外来（院外）_都道府県別薬効分類別数量.xlsx` → 都道府県別
- `【内服】外来（院外）_全国薬効分類別数量.xlsx` → 全国集計

二次医療圏別の処方薬ファイルは公開されていないことを全ファイルの調査により確認した（2026-06-22 確認）。

### 対応措置

処方薬指標（薬効396）は以下の方針で取り扱う:
1. **主解析（二次医療圏 n=335）**: 処方薬指標を除外し、5指標で実施
2. **頑健性チェック（都道府県 n=47）**: 6番目の追加指標として処方薬量を使用
3. **論文への記載**: Supplementary Table S1「処方薬データ除外理由の文書化」に明記済み
4. **コードブック**: `A3_codebook.csv` の処方薬エントリに `limitations` フィールドとして記録済み

### コードブックへの記載箇所

`02_Data/interim/A3_codebook.csv` の `antidiabetic_drug_qty_rate` エントリ:
```
"【重要】NDB No.11において処方薬データは都道府県レベルのみ公開。
二次医療圏（約335圏）レベルの集計ファイルは存在しないため、
本研究の主解析（二次医療圏）では使用しない。
都道府県レベルの頑健性チェック（n=47）でのみ追加指標として使用する。"
```

- B001-27: 13.1%（44/335圏）

**実装上の注意点**:
- `district_code`（4桁数字）をマージキーとして使用（`district_name` は複数圏で同名のため不可）
- 健診ファイルの地区コードは float64 で読み込まれるため `int → zfill(4)` 変換が必要
- 「二次医療圏判別不可」行は除外してから重複解消を行う（コード 4705「八重山」の重複問題を解決）

### タスクB: 年齢標準化（完了）

**手法**: 直接標準化法（健診指標）/ 代理分母換算（臨床指標）

| 基準人口 | 2020年国勢調査 全国 40-74歳（5歳階級別） |
|---------|--------------------------------------|
| 基準人口合計 | 58,639,000人 |

| ファイル | 内容 |
|---------|------|
| `B1_district_standardized.csv` | 二次医療圏標準化済みデータ（335圏） |
| `B2_prefecture_standardized.csv` | 都道府県標準化済みデータ（47件） |
| `B3_standardization_comparison.csv` | 標準化前後比較 |
| `B4_analysis_dataset_district.csv` | 解析用データセット（335圏・5指標） |

**標準化結果**:
- HbA1c 高値率: 粗率 7.66% → 標準化率 7.55%（わずかに低下）
- FPG 異常率: 粗率 15.53% → 標準化率 16.28%（わずかに上昇 = 若い基準人口で調整した効果）

**臨床指標の制約**:
NDB No.11 の二次医療圏別ファイルには年齢別内訳がないため、直接法の適用が不可能。
特定健診受診者数（健診 hba1c_total_n）を代理分母として使用（1,000人あたり率）。
この制約は Methods セクション「Age Standardization」に明記。
